Tutoriel Complet Python + Docker + FastAPI

Développement direct sur serveur Ubuntu via VS Code Remote SSH avec Docker, FastAPI, Nginx, et rechargement automatique des fichiers Python.

🧭 OBJECTIF FINAL

  • 💻 Développement directement sur serveur via VS Code Remote SSH
  • 🐳 Docker avec backend Python FastAPI et Nginx
  • ⚡ Rechargement automatique des fichiers Python (--reload)
  • 🔌 Docker Compose prêt à lancer facilement
  • 📝 Résumé clair et commandes expliquées

🔷 PARTIE 0 — PRÉREQUIS

Avant de commencer, assure-toi d’avoir :

  • Serveur Ubuntu accessible (IP + utilisateur + mot de passe ou clé SSH)
  • PC avec Visual Studio Code
  • Connexion Internet sur serveur et PC

🔷 PARTIE 1 — CONNEXION AU SERVEUR

1️⃣ Connexion SSH

ssh user@IP_DU_SERVEUR

Explication :
ssh → Secure Shell, pour se connecter à distance
user → ton nom d’utilisateur sur le serveur
IP_DU_SERVEUR → adresse IP du serveur Ubuntu

2️⃣ Vérifier la connexion

pwd

Explication :
pwd → print working directory, montre le dossier actuel
Important pour vérifier que tu es bien sur le serveur

🔷 PARTIE 2 — MISE À JOUR ET OUTILS DE BASE

3️⃣ Mettre à jour la liste des paquets

sudo apt update

Explication :
sudo → exécute la commande en administrateur
apt update → met à jour la liste des paquets disponibles

4️⃣ Installer les mises à jour

sudo apt upgrade -y

Explication :
apt upgrade → installe les dernières versions des paquets
-y → confirme automatiquement

5️⃣ Installer outils essentiels

sudo apt install -y git curl unzip ufw

Explication :
git → contrôle de version
curl → télécharger fichiers ou scripts depuis internet
unzip → décompresser fichiers .zip
ufw → firewall simple

🔷 PARTIE 3 — SÉCURISATION DU SERVEUR

6️⃣ Activer firewall et autoriser SSH

sudo ufw allow OpenSSH
sudo ufw allow 80
sudo ufw allow 8000
sudo ufw enable

Explication :
ufw allow OpenSSH → permet connexion SSH
ufw allow 80 → autorise HTTP
ufw allow 8000 → pour backend FastAPI
ufw enable → active firewall

7️⃣ Vérifier le statut du firewall

sudo ufw status

🔷 PARTIE 4 — INSTALLATION DOCKER

8️⃣ Installer Docker

sudo apt install -y docker.io

Explication :
docker.io → installe Docker Engine

9️⃣ Démarrer et activer Docker

sudo systemctl start docker
sudo systemctl enable docker

Explication :
systemctl start → démarre le service
systemctl enable → démarre automatiquement au reboot

Option pratique : éviter sudo

sudo usermod -aG docker $USER
newgrp docker

usermod -aG docker $USER → ajoute ton utilisateur au groupe Docker
newgrp docker → recharge la session pour appliquer le groupe

10️⃣ Tester Docker

docker run hello-world

11️⃣ Installer Docker Compose

sudo apt install -y docker-compose
docker-compose --version

Docker Compose → outil pour lancer plusieurs conteneurs ensemble

🔷 PARTIE 5 — STRUCTURE DU PROJET

12️⃣ Créer dossiers projet

mkdir -p ~/app/{backend,frontend,nginx}
cd ~/app

mkdir -p → crée plusieurs dossiers d’un coup
~/app → dossier principal

🔷 PARTIE 6 — BACKEND PYTHON (FastAPI)

13️⃣ Fichier main.py

nano backend/main.py

Colle :

from fastapi import FastAPI

app = FastAPI()

@app.get("/")
def read_root():
    return {"message": "Backend OK 🚀"}

14️⃣ Dockerfile backend pour DEV

nano backend/Dockerfile

Colle :

FROM python:3.11
WORKDIR /app
COPY . .
RUN pip install fastapi uvicorn
CMD ["uvicorn", "main:app", "--host", "0.0.0.0", "--port", "8000", "--reload"]

Explications importantes :
--reload → redémarre automatiquement FastAPI quand tu modifies un fichier
COPY . . → copie ton code dans le conteneur
RUN pip install ... → installe dépendances dans l’image

15️⃣ Docker Compose backend volume pour DEV

backend:
  build: ./backend
  container_name: backend
  volumes:
    - ./backend:/app
  ports:
    - "8000:8000"

Astuce DEV : Avec le volume, tes modifications Python sont instantanément visibles dans le conteneur → pas besoin de rebuild

🔷 PARTIE 7 — NGINX

16️⃣ Config default.conf

nano nginx/default.conf

Colle :

server {
    listen 80;

    location / {
        return 200 'Frontend à venir...';
        add_header Content-Type text/plain;
    }

    location /api/ {
        proxy_pass http://backend:8000/;
    }
}

/api/ → redirige vers backend
/ → frontend placeholder

17️⃣ Docker Compose Nginx

nginx:
  image: nginx
  container_name: nginx
  ports:
    - "80:80"
  volumes:
    - ./nginx:/etc/nginx/conf.d
  depends_on:
    - backend

volumes → remplace les fichiers conf du conteneur avec les tiens

🔷 PARTIE 8 — LANCEMENT

18️⃣ Lancer DEV serveur

docker-compose up

démarre conteneurs existants, pas besoin de rebuild si tu modifies Python

Background

docker-compose up -d

-d → détaché, terminal libre

19️⃣ Tester backend

Ouvre navigateur :

http://IP_DU_SERVEUR:8000/

→ tu dois voir : {"message":"Backend OK 🚀"}

🔷 PARTIE 9 — VS CODE REMOTE SSH

20️⃣ Installer extension

Remote - SSH

21️⃣ Config SSH

nano ~/.ssh/config

Ajoute :

Host monserveur
    HostName IP_DU_SERVEUR
    User user

22️⃣ Connexion via VS Code

Ctrl+Shift+P → Remote-SSH: Connect to Host → monserveur
Ouvre /home/user/app

🔷 ✅ POINTS IMPORTANTS DEV

  • Monte le backend avec un volume → modifications instantanées
  • Utilise --reload avec Uvicorn → pas besoin de rebuild pour chaque changement
  • --build seulement si Dockerfile ou dépendances changent
  • Nginx fonctionne automatiquement via Docker → ton fichier .conf suffit

💡 Avec ça, tu as un environnement complet, prêt pour coder et tester ton projet, sans rebuild inutile.
